# Analog Mixed Verification

## AMS0

`AMS0` містить behavioral AMS-модель простого АЦП `adc_simple_001`.

Що робить модуль:

- приймає аналоговий вхід `IN`;
- квантує його відносно `VREF`;
- вибирає значення на фронті `CLK`;
- вимагає валідних `VDD3V3`, `VREF`, `IBIAS10U` та активного `EN`;
- віддає 8-бітний код через окремі виходи `OUT0..OUT7`;
- має затримку результату на один такт.

Склад каталогу:

- `AMS0/simple_adc.ams` - базова 8-бітна модель ADC;
- `AMS0/simple_adc_001.ams` - альтернативна параметризована версія;
- `AMS0/tb_adc_simple.ams` - AMS testbench з готовими test case-ами.

### Test case-и в `tb_adc_simple.ams`

У testbench реалізовані такі сценарії:

- `TC1` - `IN = 0.0 В`, очікується код `0`;
- `TC2` - `IN = 0.5 В`, очікується `next_code` близько `64`;
- `TC3` - `IN = 1.0 В`, очікується `next_code` близько `128`;
- `TC4` - `IN = 1.5 В`, очікується `next_code` близько `191`;
- `TC5` - `IN = 2.0 В`, очікується насичення до `255`;
- `TC6` - `IN = 2.2 В`, очікується утримання saturation на `255`;
- `TC7` - `IN = -0.1 В`, очікується clamp до `0`;
- `TC8` - `EN = 0`, очікується скидання виходу в `0`;
- `TC9` - `VDD3V3 = 3.0 В`, очікується fault і нульовий код;
- `TC10` - `VREF = 1.8 В`, очікується fault і нульовий код;
- `TC11` - `IBIAS10U = 8 мкА`, очікується fault і нульовий код.

Testbench також враховує, що модель має затримку на один такт, тому перевірка виходів виконується після фронту `CLK` з невеликою затримкою.
