# **Завдання з RTL-проєктування: Модуль обробки зображення з нелінійними просторовими фільтрами 3×3 (AXI4-Stream)**

### **Загальний опис**
Розробити субмодуль фільтра зображення, який застосовує до вхідного вікна 3×3 один із чотирьох **нелінійних (порядкових) фільтрів**. Передача даних здійснюється за допомогою спрощеного протоколу **AXI4-Stream** (сигнали `TVALID`/`TREADY`/`TDATA`, без `TLAST`/`TID`/`TSTRB`).

Модуль повинен:
- Обробляти **один піксель** на основі вхідного вікна 3×3.
- Обирати тип фільтра за допомогою конфігураційного сигналу.
- Видавати оброблений піксель через вихідний AXI-Stream канал.

---

### **Конфігурація**

**1. Типи фільтрів** (обираються через `i_config_select`, 2 біти):

| `config_select` | Фільтр | Опис |
|---|---|---|
| `00` | **Min-фільтр** | Вихід = мінімальне значення серед 9 пікселів вікна |
| `01` | **Max-фільтр** | Вихід = максимальне значення серед 9 пікселів вікна |
| `10` | **Медіанний фільтр** | Вихід = медіана (5-те за порядком значення) із відсортованих 9 пікселів |
| `11` | **Оператор Собеля** | Вихід = наближення `|Gx| + |Gy|`, обчислене згідно з ядрами нижче |

Ядра оператора Собеля (для режиму `11`):

```
Gx:                      Gy:
[ -1   0  +1 ]           [ -1  -2  -1 ]
[ -2   0  +2 ]           [  0   0   0 ]
[ -1   0  +1 ]           [ +1  +2  +1 ]
```

**2. Передача даних:**
- **Вхід**: вікно пікселів 3×3 (9 значень) передається одним тактом через `S_AXIS`-подібний вхідний канал.
- **Вихід**: один оброблений піксель передається через `M_AXIS`-подібний вихідний канал.

---

### **Вимоги до дизайну**

- **Застосування фільтра**:
  - Для `00`/`01` — знайти min/max серед 9 значень (наприклад, деревом компараторів).
  - Для `10` — виконати сортувальну мережу (sorting network) на 9 елементів і взяти середній елемент.
  - Для `11` — обчислити зважені суми `Gx`, `Gy`, взяти модулі та підсумувати.
  - Нормалізувати/насичувати (saturate) результат до розрядності вхідного пікселя (`DATA_BW`), щоб уникнути переповнення.

- **Інтеграція**:
  - Використати протокол **AXI4-Stream (спрощений)** із класичним handshake `TVALID & TREADY`.

- **Конфігурація**:
  - Динамічний вибір фільтра сигналом `i_config_select` (може змінюватись між транзакціями).

> [!TIP]
> Реалізувати конвеєризацію (pipelining): порівняльне дерево / сортувальна мережа / обчислення Собеля розбити на кілька стадій, щоб перекривати обробку послідовних вікон 3×3 та зменшити критичний шлях.

---

### **Опис портів вводу/виводу**

| **Ім'я порту**            | **Напрямок** | **Опис**                                                                                     |
|---------------------------|--------------|-----------------------------------------------------------------------------------------------|
| `i_clk`                    | Вхід         | Тактовий сигнал модуля.                                                                        |
| `i_rstn`                   | Вхід         | Активний низьким рівнем асинхронний/синхронний скид, приводить модуль у початковий стан.       |
| `i_config_select`          | Вхід         | 2-бітний сигнал вибору типу фільтра (Min, Max, Медіана, Собель).                                |
| `i_axis_in_tvalid`         | Вхід         | AXI-Stream `TVALID` вхідного каналу — сигналізує про дійсність вікна 3×3.                       |
| `o_axis_in_tready`         | Вихід        | AXI-Stream `TREADY` вхідного каналу — готовність модуля прийняти вікно 3×3.                     |
| `i_axis_in_tdata`          | Вхід         | Дані вікна 3×3 (9 пікселів), кожен розрядністю `DATA_BW`, упаковані в один вектор `TDATA`.       |
| `o_axis_out_tvalid`        | Вихід        | AXI-Stream `TVALID` вихідного каналу — сигналізує про дійсність обробленого пікселя.            |
| `i_axis_out_tready`        | Вхід         | AXI-Stream `TREADY` вихідного каналу — готовність приймача прийняти оброблений піксель.          |
| `o_axis_out_tdata`         | Вихід        | Оброблений піксель розрядністю `DATA_BW`, результат застосування обраного фільтра.               |

---

### **Параметри модуля**

| **Параметр** | **Опис** |
|---|---|
| `DATA_BW` | Розрядність одного пікселя (наприклад, 8 біт) |
| `WINDOW_SIZE` | Кількість пікселів у вікні = 9 (фіксовано, 3×3) |

*(`TDATA` вхідного каналу має розрядність `9 × DATA_BW`, вихідного — `DATA_BW`.)*
